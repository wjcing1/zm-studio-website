# 🚀 快速替换图片指南

## 📂 文件夹结构

```
网站设计/
└── public/
    └── images/
        ├── home/          ← 首页4张大图放这里
        ├── projects/      ← 项目图片放这里
        ├── about/         ← 关于页图片放这里
        └── services/      ← 服务页图片放这里
```

---

## ⚡ 超简单 3 步替换首页图片

### 第 1 步：准备图片

准备 4 张横向照片，建议尺寸 2400×1600px

### 第 2 步：重命名

把图片重命名为：
- `hero1.jpg` 
- `hero2.jpg`
- `hero3.jpg`
- `hero4.jpg`

### 第 3 步：放入文件夹

把这 4 张图片拖到：`public/images/home/` 文件夹

**完成！** 刷新浏览器就能看到你的图片了！

## 🎨 图片规格建议

### 首页大图
- **尺寸**：2400×1600px 或更大
- **比例**：16:9 或 3:2（横向）
- **格式**：JPG
- **大小**：压缩后每张不超过 500KB
- **建议**：选择有视觉冲击力的横向构图

### 项目图片
- **尺寸**：1200×900px
- **比例**：4:3
- **格式**：JPG
- **大小**：每张不超过 300KB

### 其他图片
- **关于页**：1200×800px
- **服务页**：800×600px

## 💡 图片压缩工具推荐

压缩图片可以加快网站加载速度：

1. **在线工具**：
   - TinyPNG (https://tinypng.com) - 免费，效果好
   - Squoosh (https://squoosh.app) - Google 出品

2. **软件工具**：
   - Adobe Photoshop："文件 → 导出 → 存储为 Web 所用格式"
   - GIMP（免费）："文件 → 导出为"，选择 JPG 质量 80-85

## 🔧 修改图片对应的标题

打开 `src/App.tsx`，找到第 **107** 行开始的 `sections` 数组：

```typescript
{ 
  type: "image",
  title:"这里改标题",      // ← 改这里
  subtitle:"这里改副标题",  // ← 改这里
  align:"left",           // left=左对齐, right=右对齐
  url:"/images/home/hero1.jpg"
},
```

## ✅ 检查清单

在替换图片前，确保：

- [ ] 图片是横向构图（不是竖向）
- [ ] 文件名全是英文小写（不含中文、空格）
- [ ] 文件格式是 `.jpg` 或 `.png`
- [ ] 图片已经压缩过（文件不超过 500KB）
- [ ] 放在了正确的文件夹里
- [ ] 文件名完全一致（注意 `.jpg` 不是 `.JPG`）

## 🎯 示例：完整替换流程

**假设你要替换首页第1张图片：**

1. ✅ 选择一张照片，比如 `我的展览照片.jpg`
2. ✅ 重命名为 `hero1.jpg`（全小写）
3. ✅ 用工具压缩到 500KB 以内
4. ✅ 拖到 `public/images/home/` 文件夹
5. ✅ 刷新浏览器（http://localhost:5173）
6. ✅ 看到图片已更新！

## ❓ 常见问题

### Q1: 图片不显示，显示的是灰色方块？
**A**: 检查：
- 文件名是否完全一致（注意大小写）
- 图片是否真的在 `public/images/home/` 文件夹里
- 刷新浏览器（按 F5）

### Q2: 图片显示了但是被裁切了？
**A**: 这是正常的，图片会自动适应屏幕大小。建议使用横向构图的照片，重要内容放在中间。

### Q3: 可以用 PNG 格式吗？
**A**: 可以！但文件会比 JPG 大很多。如果用 PNG，确保压缩到合理大小。

### Q4: 我只想改第1张图片，其他保留？
**A**: 可以！只替换 `hero1.jpg`，其他图片保持在线链接或不动即可。

### Q5: 图片文件名必须是 hero1、hero2 吗？
**A**: 这样最方便。如果想用自定义文件名，需要修改 `src/App.tsx` 中的 `url` 路径。
